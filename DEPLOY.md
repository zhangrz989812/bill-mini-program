# 微信小程序记账本 - 部署指南

## 部署前准备

### 1. 环境要求
- 微信开发者工具最新版本
- 微信小程序账号（已认证）
- 云开发环境（已开通）

### 2. 项目配置检查
在部署前，请确认以下配置：

#### AppID配置
1. 打开 `project.config.json`
2. 修改 `appid` 为你的小程序AppID
3. 保存文件

#### 云开发环境配置
1. 打开 `app.js`
2. 修改 `wx.cloud.init` 中的 `env` 参数
3. 设置为你的云开发环境ID

### 3. 云函数部署
云函数需要单独部署到云开发环境：

#### 部署用户登录云函数
1. 在微信开发者工具中，展开 `cloud/functions/user-login`
2. 右键点击 `user-login` 文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待部署完成

#### 部署添加记录云函数
1. 展开 `cloud/functions/add-record`
2. 右键点击 `add-record` 文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待部署完成

### 4. 数据库集合创建
云数据库需要手动创建集合：

#### 创建用户集合 (users)
1. 打开云开发控制台
2. 进入"数据库"
3. 点击"+"创建集合
4. 输入集合名称：`users`
5. 点击确定

#### 创建记账记录集合 (records)
1. 重复上述步骤
2. 集合名称：`records`
3. 设置权限："仅创建者可读写"

#### 创建分类集合 (categories)
1. 重复上述步骤
2. 集合名称：`categories`
3. 设置权限："仅创建者可读写"

#### 创建预算集合 (budgets)
1. 重复上述步骤
2. 集合名称：`budgets`
3. 设置权限："仅创建者可读写"

### 5. npm构建
1. 在微信开发者工具中：工具 → 构建npm
2. 勾选"使用npm模块"
3. 等待构建完成

## 正式发布

### 1. 版本管理
1. 修改 `app.json` 中的版本号（如有）
2. 更新 `README.md` 中的版本信息
3. 提交代码到版本库

### 2. 上传代码
1. 在微信开发者工具中：项目 → 上传
2. 填写版本号（如：1.0.0）
3. 填写项目备注
4. 点击"上传"

### 3. 提交审核
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"管理" → "版本管理"
3. 找到刚上传的版本
4. 点击"提交审核"
5. 填写审核信息：
   - 功能介绍：个人记账小程序，支持收入支出记录、分类管理、统计分析、预算管理
   - 截图：上传小程序界面截图
   - 测试账号：如有需要，提供测试账号

### 4. 审核注意事项
- 确保小程序内容符合微信规范
- 不涉及敏感内容
- 功能完整可用
- 用户隐私保护到位

### 5. 发布上线
- 审核通过后，在版本管理页面点击"发布"
- 选择"全量发布"或"分阶段发布"
- 等待发布完成

## 常见问题

### Q: 云函数部署失败怎么办？
A: 检查网络连接，确认云开发环境已开通，尝试重新部署。

### Q: 数据库权限如何设置？
A: 在云开发控制台 → 数据库 → 集合 → 权限设置，选择合适的权限规则。

### Q: 如何查看云函数日志？
A: 在云开发控制台 → 云函数 → 选择函数 → 日志。

### Q: 真机预览数据不同步？
A: 确认使用的是同一个微信账号，检查云开发环境配置。

## 性能优化建议

### 1. 数据库优化
- 为常用查询字段创建索引
- 避免频繁的数据库操作
- 使用聚合查询提高效率

### 2. 前端优化
- 使用分页加载数据
- 图片压缩和懒加载
- 避免不必要的重绘

### 3. 云函数优化
- 合并相似的云函数
- 使用异步操作
- 设置合理的超时时间

## 安全建议

### 1. 数据安全
- 设置合适的数据库权限
- 定期备份重要数据
- 不存储敏感信息

### 2. 代码安全
- 不在前端暴露敏感信息
- 验证所有用户输入
- 防止SQL注入和XSS攻击

### 3. 用户隐私
- 遵守微信隐私规范
- 明确告知用户数据用途
- 提供数据删除功能
